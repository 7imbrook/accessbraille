ABKeyboard Programming Guide

Overview

Access Braille is a braille accessibility framework for iPad that creates an interface to type in braille on top of an application. The project focuses on providing a more efficient and accessible method of typing on an iOS device than the current defaults provided by Apple. We hope to inevitably facilitate device use for the visualy impaired and blind. This document focuses on the usage of framework in your own application.

The AccessBraille ABKeyboard.framework provides you with a keyboard interface along with other classes that help with handling output from the keyboard. The keyboard can be used two ways. You can implement the delegate calls that recieve keystrokes and handling typing yourself. Or you can pass the keyboard a UITextView and the keyboard will handle responder calls and act as the first responder.

At a Glance

There are two ways of interacting with the keyboard, using the delegate calls and handling typing yourself, or passing a referance to a UITextView and lettting the ABKeyboard become the first responder for the view.

  • To handle keystokes yourself override the methods found in the ABKeyboard protocol reference.
  • To let the keyboard use a UITextView use -setOutput: on your ABKeyboard instance.
To create a keyboard you need to instanciate ABKeyboard with a delegate, -initWithDelegate:. This will add a gesture recognizer to the delegate used for activating the keyboard. The delegate must be a subclass of UIView and follow the ABKeyboard protocol.

Structure

The ABKeyboard framework provides an overlay that sits ontop of your applications UI. When the keyboard is active the ABTouchLayer is added to your view hierarchy and brought to the front. It takes over all touch input.
